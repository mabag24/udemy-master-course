Block tags

  • <aside>

    Information that is related to, but seperate from the other content on a page

  • <article>

    A standalone piece of content that is publishable by itself e.g. a blog article

  • <section>

    A section of related content that can logically be grouped together, used for outlining purposes

  • <figure> & <figcaption>

    Groups related content together but seperately from the main content flow

  • <address>

    Contact Information

  • <nav>

    Groups together navigation elements

Inline Tags

  • <abbr>

    displays text on hover to explain abbreviations or acronyms

  • <kbd>

    informs user that the wrapped text should be a keyboard input

  • <blockquote> & <q>

    informs user that the text is a quote

  • <sup>

    raises text for indices etc

  • <sub>

    lowers text for logs etc

  • <em>

    Used to emphasise text (typically displays in italics)

CSS BOX MODEL

  • Content: Text, images, etc.
  • Border: A line around the element, still inside the element
  • Padding: Invisible space around the content, indside the element
  • Margin: Space outside of the element, between elements
  • Fill Area: Area that gets filled with background color or background image

CSS Display

  • Display: Block

    Makes an element span the length of the page (makes the same as a full size element). Starts on a new line and can be affected by height and weight properties. Displays vertically one after another

  • Display: inline

    Displays the element as an inline property (like span), cannot be manipulated by height/weight properties. Causes no line breaks. Only left and right margins and paddings apply

  • Display: flex

    Displays element as a flex container see flexbox section

  • Display: Grid

    Displays an element as a grid container. see grid section

  • Display: table

    lets the element behave like a <table> element

  • Display: inline- ______

    lets the element be displayed as an inline-level property, e.g inline-table or inline-grid

CSS Positioning Properties

  • Position: relative

    Elements are "in flow" - laid out according to their order in the HTML code.

  • Position: absolute

    Element is considered "out of flow". Doesn't have an impact on surrounding elements and may overlap them. Top, bottom, left or right can be used to offset the element. Set a parent element to relative to make the absolute element compared to it.
